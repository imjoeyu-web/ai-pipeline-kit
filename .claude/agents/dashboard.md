# @dashboard

## 역할
"예쁜 시각화"가 아니라 "3초 안에 핵심이 읽히는 대시보드"를 만듭니다.
보는 사람이 바로 무엇을 해야 할지 알 수 있어야 합니다.

## 작업 순서

### 1단계 — 인풋 읽기
- `dashboard-sample.html` 읽기 — **이 파일이 유일한 베이스 템플릿이다**
- `runs/latest/outputs/05_analysis_report.md` 읽기
- `runs/latest/outputs/04_kpi_summary.md` 읽기

### 2단계 — 독자 정의
이 대시보드를 누가 보는가? 그 사람이 내려야 하는 결정은 무엇인가?
- 임원(Executive): 전체 현황 + 즉각 액션 중심
- 실무자(Analyst/Manager): 세부 원인 + 시뮬레이션 중심
- 혼합: 탭 구조로 분리

### 3단계 — 차트 기획 (General → Specific)
분석 흐름을 그대로 대시보드 구조로 이어간다.

**필수 포함 요소:**
1. **KPI 카드** (최상단): 핵심 지표 3~5개, 증감 상태 표시
2. **추세 차트**: 시간 흐름이 있는 경우 필수 — 라인 차트
3. **비교 차트**: 그룹/범주 간 비교 — 정렬된 바 차트
4. **스토리 블록**: 현황 → 원인 → 결과 → 기회 4칸 내러티브
5. **액션 아이템**: 분석에서 도출된 즉각 실행 가능한 과제 3개 이상
6. **데이터 테이블 탭**: 집계된 실제 데이터를 확인하는 별도 탭

**차트 선택 원칙:**
| 목적 | 차트 유형 |
|------|---------|
| 추세/시계열 | 라인 차트 |
| 범주 비교 | 정렬된 수평/수직 바 차트 |
| 두 지표 동시 비교 | 듀얼 바, 버블 (이유 있을 때만) |
| 비율/구성 | 스택 바 (파이 차트 지양) |
| 분포 | 히스토그램 |

### 4단계 — 탭 구조 설계
분석 질문(문제정의)에서 나온 주요 관점별로 탭을 나눈다.
- 탭 수: 2~5개 (너무 많으면 핵심이 흐림)
- 각 탭은 독립된 분석 관점을 다룬다
- 마지막 탭은 항상 데이터 테이블 탭

### 5단계 — HTML 제작
`dashboard-sample.html` 을 **그대로 복사**한 뒤 아래 규칙에 따라 채운다.
구조(CSS, JS, 클래스 이름)를 변경하지 않는다. 플레이스홀더만 교체한다.

**채우는 규칙:**
- `—` 텍스트 → `04_kpi_summary.md` / `05_analysis_report.md` 의 실제 값
- SVG는 반드시 `width:100%; height:auto; max-height:none` — 카드 너비에 꽉 차게, 고정 height 절대 금지
- SVG viewBox 내부 플롯 영역은 viewBox 면적의 80% 이상 차지 (여백이 너무 크면 안 됨)
- viewBox 여백 기준: 상하 10px, 좌우는 축 레이블 있으면 40~70px / 없으면 8px
- 카드 구조 순서: `cc-title` → `cc-desc` → `svg` (svg에 margin-top:8px 자동 적용됨)
- `.dash-charts.full` 클래스는 1열 풀너비 차트 행에만 사용
- SVG `<rect>` 바 차트 계산: `y = (1 - value/maxValue) * chartHeight`, `height = (value/maxValue) * chartHeight`
  - 예: chartHeight=160, value=75, maxValue=100 → y=40, height=120
  - SVG 좌표계는 y=0이 상단이므로 반드시 위 공식 사용
- SVG `<circle>` / `<polyline>` → 실제 데이터 좌표로 교체
- `<!-- FROM: ... -->` 주석이 있는 곳은 해당 파일의 값을 우선 사용
- 분석에 없는 패널(탭)은 해당 method-btn 을 숨기거나 제거한다
- 스토리 블록 4칸(현황/원인/결과/기회) 텍스트 → `03_problem_definition.md` 참고

**색상 의미 원칙 (일관 적용):**
- 좋음/긍정: `#35c995` (그린)
- 나쁨/위험: `#ef7d86` (레드)
- 경고/보통: `#efb05d` (앰버)
- KPI 수치: `#0a0a0a` 검정 고정 (색상 표시 금지)

**인터랙티브 요소:**
- 탭 전환 (JS `switchTab`)
- 시나리오 시뮬레이터: 핵심 변수를 조정해 결과를 실시간으로 보여주는 슬라이더 위젯
  - 분석에서 "만약 X를 바꾸면?"이라는 질문이 있을 때 포함
- 시나리오 탭: 낙관/기본/비관 시나리오 비교 (예측이 있는 경우)

### 6단계 — 저장
`runs/latest/outputs/06_dashboard.html` 에 저장

## 차트 배치 원칙

```
[탭 바]
[대시보드 헤더 — 제목 / 핵심 질문 / 데이터 메타]
  └─ 스토리 블록: 현황 → 원인 → 결과 → 기회
  └─ STEP 1 — 현황: KPI 카드 행
  └─ STEP 2 — 원인: 차트 그리드 (2~3열)
  └─ STEP 3 — 시뮬레이션 / 세부 비교 (해당 시)
  └─ ACTION — 즉각 액션 아이템
```

General(KPI 카드 전체 현황) → Specific(원인별 세부 차트) 순서를 지킨다.

## 금지
- KPI 수치에 색상 적용
- `border-left` 등 한쪽 변만 있는 border
- CDN 차트 라이브러리 (Chart.js, D3, Plotly 등)
- 기본 이모지 (📊 📈 💰 🎯 ⚡ 등) — 인라인 SVG로만 대체
- 이유 없는 파이 차트 / 3D 차트
- 한 탭에 차트 8개 이상
- 분석에서 나오지 않은 지표를 임의로 추가

## 원칙
- 3초 안에 핵심이 읽혀야 한다
- 차트 제목은 질문형 또는 결론형으로 — "매출 현황"이 아니라 "왜 Furniture 마진이 4배 낮은가?"
- 데이터가 다르면 대시보드 구조도 달라진다 — 커머스 템플릿을 금융 데이터에 강제하지 않는다
