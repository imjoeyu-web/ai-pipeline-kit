## 수행한 전처리

전처리 없음. 아래 이유로 @eda가 원본 그대로 수신한다.

- **Saving accounts / Checking account의 "NA"**: 실제 결측인지, "계좌 없음"이라는 유효한 범주인지 도메인 판단이 필요. EDA 단계에서 분포 확인 후 결정
- **Credit amount 이상치**: 극단값이지만 대출 데이터에서 자연스러운 범위 — 제거 근거 없음
- **Job 정수형**: 범주 의미 보존을 위해 형 변환 없이 그대로 전달

## @eda 에게 전달하는 주의사항

1. **Checking account (39.4% NA)** — 단순 결측 처리 전, "계좌 없음(no checking account)" 범주로 해석하는 가설을 우선 검토. 이 해석이 맞다면 정보가 풍부한 유효 범주가 됨
2. **Saving accounts (18.3% NA)** — 동일한 접근 권장
3. **타깃 컬럼 없음** — 분석 방향을 클러스터 발굴 또는 세그먼트 특성 비교로 설정할 것. "신용 위험 예측"은 이 데이터만으로 불가
4. **Job 컬럼** — 0~3은 순서형. 평균 계산 또는 연속형으로 취급 금지
5. **Purpose 희귀 범주** — domestic appliances, vacation/others는 시각화 시 "기타"로 묶는 것 검토
6. **표본 크기 1,000건** — 세부 교차 분석(예: Sex × Job × Purpose) 시 셀당 표본 20건 미만 구간 발생 가능. 반드시 n 명시
